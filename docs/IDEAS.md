# Product Ideas

These ideas preserve the official-source-only data policy while making the
archive more useful for UFO/UAP readers and researchers.

## High-Value Ideas

- Release diff view: show exactly which records appeared in each sync, with
  official URLs, file types, and bundle membership.
- File clusters: expand the relationship graph into topic clusters such as
  agencies, locations, time periods, and recurring document series.
- Reader queues: let users save interesting official files locally in the
  browser for later review without accounts or server storage.
- Case annotations: support private browser-local notes that never enter the
  repo or public data.
- Bundle explorer: for large official ZIP releases, show file names, sizes, and
  source bundle metadata before users download multi-gigabyte archives.
- Provenance inspector: show how each field was derived: live page, manifest
  fallback, official bundle directory, or maintainer-normalized metadata.
- Timeline comparison mode: compare incident chronology with release chronology
  so delayed declassification patterns are easy to see.
- Map confidence states: distinguish precise coordinates, approximate regional
  coordinates, and unknown locations in the map and case metadata.

## Guardrails

- Do not ingest unofficial case facts from mirrors, social media, or community
  catalogs.
- Keep user notes private and browser-local unless a maintainer explicitly
  designs a privacy model.
- Avoid sensational labels. Use factual relationship reasons and source links.
